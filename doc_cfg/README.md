In many crates' documentation on docs.rs, such as `tokio`, `crossbeam` and so on, we can see such hints:
```text
pub struct Foo;
Available on crate feature foo only.
```

This is gnerated by feature gate `doc_cfg`.

You can enable it with `#![cfg_attr(docsrs, feature(doc_cfg))]` just like in `lib.rs`.

And docs.rs uses nightly Rust to build documentation. 
And before nightly 1.92, it's generated by `doc_auto_cfg`, after 1.92, it's merged into `doc_cfg`.

So just use `doc_cfg`, or an error will be raised.
```text
error[E0557]: feature has been removed
 --> doc_cfg/src/lib.rs:1:29
  |
1 | #![cfg_attr(docsrs, feature(doc_auto_cfg))]
  |                             ^^^^^^^^^^^^ feature has been removed
  |
  = note: removed in 1.92.0; see <https://github.com/rust-lang/rust/pull/138907> for more information
  = note: merged into `doc_cfg`

error: Compilation failed, aborting rustdoc

For more information about this error, try `rustc --explain E0557`.
```

And do not forget add this to Cargo.toml, or some cfg hints may be missed:
```toml
[package.metadata.docs.rs]
all-features = true
```

To check the result, run `cargo docs-rs -p doc_cfg --open`.
